# S3 / Object Storage

h5cpp exposes the HDF5 Read-Only S3 Virtual File Driver (ROS3 VFD) through the
`h5::ros3` property type.  Once a file is open, all normal h5cpp I/O calls work
unchanged — no S3-specific code is needed beyond the `h5::open` call.

## Prerequisites

HDF5 must be built with ROS3 support.  Check with:

```bash
h5cc -showconfig | grep ROS3
# or
python3 -c "import h5py; print(h5py.version.info)"
```

On Ubuntu the `libhdf5-dev` package already includes it (`H5_HAVE_ROS3_VFD 1` in
`H5pubconf.h`).  If compiling HDF5 from source, add `--enable-ros3-vfd` and ensure
`libcurl` and `libssl` are present.

h5cpp detects availability at configure time and sets `H5CPP_HAVE_ROS3_VFD`:

```bash
cmake -S . -B build
# → H5CPP: ROS3 VFD detected — S3 read-only support enabled
```

## Capability check at runtime

```cpp
#include <h5cpp/all>

if constexpr (h5::have_ros3_vfd)
    std::cout << "S3 read support available\n";
```

## Opening a file

### Public bucket (no credentials)

```cpp
#include <h5cpp/all>

auto fd = h5::open("s3://hdf5ros3.s3.amazonaws.com/h5ex_d_alloc.h5",
                   H5F_ACC_RDONLY,
                   h5::ros3{});
```

### Authenticated — long-term AWS credentials

```cpp
auto fd = h5::open("s3://my-bucket/run42.h5",
                   H5F_ACC_RDONLY,
                   h5::ros3{true, "us-east-1",
                            "AKIAIOSFODNN7EXAMPLE",
                            "wJalrXUtnFEMI..."});
```

Never hard-code credentials.  Read them from environment variables or a
credential file:

```cpp
auto fd = h5::open(path, H5F_ACC_RDONLY,
    h5::ros3{true,
             std::getenv("AWS_REGION"),
             std::getenv("AWS_ACCESS_KEY_ID"),
             std::getenv("AWS_SECRET_ACCESS_KEY")});
```

### Temporary credentials — STS / IAM role (HDF5 ≥ 1.12.1)

```cpp
#if H5_VERSION_GE(1,12,1)
auto fd = h5::open("s3://secure-bucket/data.h5",
                   H5F_ACC_RDONLY,
                   h5::ros3{true, region, key_id, secret, session_token});
#endif
```

## Reading data

After `h5::open`, use standard h5cpp calls:

```cpp
auto temps  = h5::read<std::vector<double>>(fd, "/sensor/temperature");
auto matrix = h5::read<arma::mat>(fd, "/results/covariance");
auto ds     = h5::open(fd, "/large/dataset");
h5::read(ds, buffer.data(), h5::offset{0}, h5::count{1024});
```

## Limitations

- **Read-only.**  `H5F_ACC_RDWR` will be rejected by the driver.
- **No partial writes.**  Append and write operations are not supported over ROS3.
- **Latency.**  Each HDF5 metadata operation may issue one or more HTTP requests.
  For scan-heavy workloads, prefer downloading the file first.
- **No write-capable S3 access** is available in this release.  That requires
  the HDF5 VOL connector layer (REST VOL + HSDS), which is tracked separately.
